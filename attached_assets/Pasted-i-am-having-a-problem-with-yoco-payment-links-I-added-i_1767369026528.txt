I understand the problem 👍
This is normal behaviour for Yoco payment links, and it’s not a bug in your code.

Why this is happening

The link you shared (pay.yoco.com/r/...) is a single-use payment request.

Once:

Someone pays successfully

The payment is completed

That specific link becomes “paid” and will always show Payment Successful.
It cannot be reused for another customer.

So even if you refresh or embed it again on your website, it will stay completed.

❌ What you cannot do

You cannot reuse the same Yoco payment link for multiple customers

You cannot reset a paid link

You cannot make it reusable via code

🧱 Basic setup (simple + works on Replit)
1️⃣ Backend (Node.js on Replit)

Create a file called server.js

import express from "express";
import fetch from "node-fetch";

const app = express();
app.use(express.json());

const YOCO_SECRET_KEY = process.env.YOCO_SECRET_KEY;

app.post("/create-payment", async (req, res) => {
  const { amount } = req.body;

  try {
    const response = await fetch(
      "https://payments.yoco.com/api/checkouts",
      {
        method: "POST",
        headers: {
          "Authorization": `Bearer ${YOCO_SECRET_KEY}`,
          "Content-Type": "application/json",
        },
        body: JSON.stringify({
          amount: amount * 100, // cents
          currency: "ZAR",
          successUrl: "https://your-site.replit.app/success",
          cancelUrl: "https://your-site.replit.app/cancel",
        }),
      }
    );

    const data = await response.json();
    res.json({ redirectUrl: data.redirectUrl });
  } catch (error) {
    res.status(500).json({ error: "Payment failed" });
  }
});

app.listen(3000, () => console.log("Server running"));
